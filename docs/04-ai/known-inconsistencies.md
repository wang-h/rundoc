# 已知不一致

记录跨文档冲突与待人工确认项，防止矛盾信息同时存在于文档站中。

## 一致性检查规则

RunDoc 的 `check` 命令会扫描以下不一致类型：

- **术语冲突**：同一概念在不同文档中使用不同名称
- **流程矛盾**：两个文档对同一流程描述不同步骤
- **过期引用**：文档引用了已删除或已重命名的文件/接口
- **版本错位**：文档记录的版本号与实际代码版本不匹配

## 处理流程

1. `rundoc check` 发现不一致项
2. 自动生成冲突报告到 `.rundoc/reports/`
3. 人工确认后，更新相关文档解决冲突
4. 无法即时解决的可标记为"已知不一致"，记录在本页
