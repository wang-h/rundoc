# 业务流程说明

记录业务流程、角色分工和交付边界。

## 文档维护流程

```text
代码变更 → 扫描感知 → 影响分析 → 文档补丁 → 人工审核 → 合并发布
```

## 角色分工

| 角色 | 职责 | 涉及文档 |
|------|------|----------|
| 产品经理 | 定义功能边界、业务规则 | `01-product/` |
| 开发工程师 | 实现功能、更新 API 文档 | `03-technical/` |
| 技术负责人 | 架构决策、技术路线 | `05-decisions/` |
| 运维工程师 | 部署配置、回滚路径 | `06-ops/` |

## 交付边界

- 代码合入主分支时，对应文档更新应在同一 MR 或关联 MR 中提交
- 文档更新不阻塞代码发布，但应在发布后 1 个工作日内补齐
- AI 上下文文档（`04-ai/`）由引擎自动刷新，无需人工维护
