# 决策记录

记录关键产品、技术与流程决策，保证团队上下文不丢失。

## 记录格式

每条决策应包含以下字段：

```text
日期：YYYY-MM-DD
决策人：姓名/角色
背景：为什么需要做这个决策
选项：考虑过哪些方案
结论：最终选择及原因
影响：对哪些文档或模块有影响
```

## 决策类型

- **产品决策**：功能范围、优先级、交互方案
- **技术决策**：架构选型、工具替换、API 兼容策略
- **流程决策**：发布节奏、文档规范、审核机制

## 示例

```md
## 2026-05-20：文档站采用 HashRouter

- 决策人：技术负责人
- 背景：静态部署不支持服务端路由重写
- 选项：HashRouter / BrowserRouter + 重写规则
- 结论：选择 HashRouter，部署最简单，无需服务器配置
- 影响：所有路由使用 # 前缀
```
