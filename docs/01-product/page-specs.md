# 页面规格

记录产品页面行为与边界，随代码变更持续更新。

## 首页（`/`）

- **路由**：`/`
- **用途**：产品入口，展示 RunDoc 能力与导航入口
- **内容区块**：
  - Hero 区：一句话定义 + 核心行动按钮
  - 能力面板：RunDoc 能做什么
  - 角色入口：按用户角色组织阅读路径
  - 文档分区：按导航分类展示文档链接

## 文档页（`/docs/*`）

- **路由**：`/docs/**`
- **用途**：渲染 Markdown 文档内容
- **布局**：左侧导航 + 中间文档正文 + 右侧目录（响应式隐藏）
- **组件**：
  - 文档正文区：标题、摘要、正文内容
  - 目录组件：自动提取 h2/h3 标题，滚动高亮
  - 翻页导航：上一篇/下一篇文档链接

## 应用页面（SPA）

以下页面属于 RunDoc 的单页应用功能，支持项目的文档协作与编辑。

### 项目列表页（`/projects`）

- **路由**：`/projects`
- **用途**：展示用户可访问的所有项目，作为文档协作的入口
- **布局**：顶部导航 + 项目卡片网格
- **组件与交互**：
  - 项目卡片：显示项目名称、描述、文档数量、状态标签（进行中/已归档）
  - 新建项目按钮：点击弹出创建项目模态框
  - 创建项目模态框：输入项目名称、描述、Git 仓库地址（可选）、项目状态
  - 点击卡片进入项目详情页
- **数据来源**：`GET /api/projects`

### 项目详情页（`/projects/:id`）

- **路由**：`/projects/:projectId`
- **用途**：查看项目信息，管理项目内的文档
- **布局**：项目头部信息 + 文档树（左侧）+ 内容区（右侧）
- **组件与交互**：
  - 项目头部：项目名称、描述、Git 仓库链接、状态
  - 文档树（DocTree）：以层级结构展示项目内所有文档，支持展开/折叠
  - 新建文档按钮：在项目中创建新文档，弹窗输入标题和父文档（可选）
  - 删除项目按钮：确认后删除项目及其所有文档
  - 点击文档节点进入文档查看页
- **数据来源**：`GET /api/projects/:id`、`GET /api/projects/:id/docs`

### 文档查看页（`/projects/:id/docs/:docId`）

- **路由**：`/projects/:projectId/docs/:docId`
- **用途**：查看文档的渲染内容与版本历史
- **布局**：面包屑导航 + 文档正文（中间）+ 版本时间线侧边栏（右侧）
- **组件与交互**：
  - 面包屑：项目名称 > 文档标题
  - 文档正文区：Markdown 渲染后的富文本内容（标题、表格、代码高亮等）
  - 编辑按钮：跳转到文档编辑页
  - 删除按钮：确认后删除当前文档
  - 版本时间线侧边栏：以时间线形式展示文档的所有历史版本（自动保存的快照），点击可查看历史版本内容
- **数据来源**：`GET /api/docs/:id`、`GET /api/docs/:id/versions`

### 文档编辑页（`/projects/:id/docs/:docId/edit`）

- **路由**：`/projects/:projectId/docs/:docId/edit`
- **用途**：编辑文档内容，支持 Markdown 实时预览
- **布局**：标题输入框 + 编辑器工具栏 + 编辑/预览区域 + 底部操作栏
- **组件与交互**：
  - 标题输入框：编辑文档标题
  - Markdown 编辑器（@runos/editor-markdown，基于 TipTap 3）：
    - 工具栏：粗体、斜体、标题层级、无序/有序列表、代码块、表格、链接、图片
    - 编辑/预览标签切换：编辑模式和渲染预览模式一键切换
  - 底部操作栏：
    - 保存草稿：保存当前内容，标记为草稿状态
    - 发布：保存并发布文档（标记为已发布状态）
    - 取消：放弃编辑，返回文档查看页
  - 键盘快捷键：`Ctrl+S` / `Cmd+S` 快速保存
- **数据来源**：`GET /api/docs/:id`（加载）、`PUT /api/docs/:id`（保存）、`POST /api/docs/:id/publish`（发布）

## 页面交互

- 搜索快捷键：`⌘K` 全局搜索
- 侧边栏：移动端抽屉模式，点击遮罩关闭
- 语言切换：支持中文/英文切换
- 响应式：980px 以下隐藏右侧目录，720px 以下隐藏顶部导航
